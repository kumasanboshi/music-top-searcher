# /design-review - 設計レビュー

コードベースのアーキテクチャと設計をレビューします。

## 使い方
```
/design-review
/design-review src/services
```

## レビュー観点

### 1. アーキテクチャ
- ディレクトリ構造の適切さ
- 責務の分離（SRP）
- 依存関係の方向

### 2. 技術的負債
- 重複コード
- 過度な複雑さ
- TODO/FIXMEの蓄積

### 3. パターン・規約
- 既存パターンとの一貫性
- 命名規則の遵守
- 型定義の適切さ

### 4. 拡張性・保守性
- 変更容易性
- テスタビリティ
- エラーハンドリング

## 出力形式

- ...

### 改善提案
| 優先度 | 箇所 | 内容 |

- ...
